# Assessment of the Driver's Condition

Проект предназначен для оценки состояния водителя по видеопотоку с веб-камеры. Основная задача — обнаруживать признаки усталости: закрытые глаза, зевоту, кивание головой и длительные периоды отсутствия лица в кадре.

В репозитории реализованы два подхода:

1. **Гибридная система** — объединяет CNN-модель и биометрические признаки лица.
2. **Baseline-система** — использует только CNN-модель без биометрии и калибровки.

Также доступен скрипт для сравнения двух методов в режиме side-by-side и записи результатов в CSV.

## Возможности

### Гибридная система

Гибридный метод использует два источника признаков:

- изображение лица, подаваемое в ONNX-модель;
- биометрические признаки, извлекаемые с помощью MediaPipe Face Mesh.

Извлекаемые биометрические признаки:

- **EAR** — Eye Aspect Ratio, показатель открытия глаз.
- **MAR** — Mouth Aspect Ratio, показатель открытия рта.
- **Yaw/Pitch** — приблизительная оценка поворота и наклона головы.
- **PERCLOS** — доля времени с закрытыми глазами в скользящем окне.
- Длительность событий:
  - микросон;
  - зевота;
  - кивание головой;
  - отсутствие лица в кадре.

Гибридная система выполняет начальную калибровку пользователя. Во время калибровки собираются персональные пороги для EAR, MAR, положения головы и смещения CNN-скоринга.

### Baseline-система

Baseline-система использует только CNN-модель `baseline_cnn.onnx`, принимающую изображение лица. Она не использует MediaPipe, биометрические признаки и персональную калибровку.

## Состояния системы

Проект использует следующие состояния:

| Состояние              | Значение                                 |
| ---------------------- | ---------------------------------------- |
| `ACTIVE`               | Нормальное состояние                     |
| `TIRED`                | Лёгкая усталость                         |
| `DROWSY`               | Сонливость                               |
| `!!! WAKE UP !!!`      | Критическое предупреждение               |
| `NO FACE / DISTRACTED` | Лицо не обнаружено или водитель отвлёкся |
| `CALIBRATING`          | Идёт калибровка гибридной системы        |
| `MODEL ERROR`          | Ошибка загрузки модели                   |

## Установка

Проект использует Python 3.12 и зависимости из `pyproject.toml`.

Рекомендуемый способ установки — через `uv`:

```bash
uv sync
```

Если `uv` не установлен, можно использовать стандартный `pip`:

```bash
python -m venv .venv
source .venv/bin/activate   # Linux/macOS
# или
.venv\Scripts\activate      # Windows

pip install .
```

Основные зависимости:

- `opencv-python`
- `numpy`
- `mediapipe`
- `onnxruntime`

## Запуск сравнения hybrid и baseline

Основной скрипт сравнения:

```bash
uv run python compare.py
```

или из активированного виртуального окружения:

```bash
python compare.py
```

Скрипт открывает веб-камеру `0` и запускает две системы одновременно:

- слева — гибридная калиброванная система;
- справа — baseline CNN.

Результаты записываются в файл:

```text
comparison_results.csv
```

### Горячие клавиши

| Клавиша | Действие                          |
| ------- | --------------------------------- |
| `q`     | Выход                             |
| `1`     | Показать только hybrid            |
| `2`     | Показать только baseline          |
| `b`     | Показать обе системы side-by-side |
| `s`     | Пауза / продолжение               |
| `r`     | Перекалибровать гибридную систему |

## Результаты сравнения по трём людям

Среднее распределение состояний по данным трёх участников:

| Состояние | Гибридная модель | Базовая модель |
| --------- | ---------------- | -------------- |
| `ACTIVE`  | 63.5%            | 25.3%          |
| `TIRED`   | 21.4%            | 35.4%          |
| `DROWSY`  | 15.1%            | 39.3%          |

Интерпретация: гибридная модель чаще классифицирует состояние как нормальное (`ACTIVE`) и реже переводит его в `DROWSY`, чем базовая CNN-модель. Это согласуется с ожидаемым эффектом калибровки: персональные пороги и биометрические признаки уменьшают количество ложных срабатываний.

## Обучение модели

Скрипт `код_обучения.py` предназначен для запуска в Google Colab. Он выполняет:

1. Конвертацию YOLO-датасета в классификационный датасет.
2. Извлечение биометрических признаков через MediaPipe.
3. Кэширование биометрии для ускорения обучения.
4. Обучение модели `MobileNetV3BiometricFusion`.
5. Оценку качества на тестовом наборе.
6. Экспорт обученной модели в ONNX.

Входной датасет должен быть организован в формате YOLO:

```text
dataset/
├── train/
│   ├── images/
│   └── labels/
├── valid/
│   ├── images/
│   └── labels/
└── test/
    ├── images/
    └── labels/
```

Модель обучается для двух классов:

| Класс     | Описание               |
| --------- | ---------------------- |
| `active`  | Активное состояние     |
| `fatigue` | Усталость / сонливость |

После обучения модель экспортируется в файл:

```text
mobilenet_bio_fatigue.onnx
```

## Модели

### `mobilenet_bio_fatigue.onnx`

Гибридная модель принимает два входа:

1. Изображение лица размером `224x224`, нормализованное как ImageNet.
2. Вектор биометрических признаков размером `8`.

Выход модели — логиты двух классов:

```text
[active_logit, fatigue_logit]
```

### `baseline_cnn.onnx`

Baseline-модель принимает один вход:

1. Изображение лица размером `224x224`, нормализованное как ImageNet.

Выход модели — логиты двух классов:

```text
[active_logit, fatigue_logit]
```

## Важные замечания

- Для корректной работы требуется веб-камера, доступная как `VideoCapture(0)`.
- Перед использованием гибридной системы необходимо пройти калибровку.
- В проекте используются предварительно подготовленные ONNX-модели. Если они отсутствуют, нужно обучить и экспортировать модель через `код_обучения.py`.
- `код_обучения.py` рассчитан на выполнение в Google Colab и использует Google Drive для сохранения модели и результатов.
